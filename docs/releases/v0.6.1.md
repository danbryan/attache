Attaché 0.6.1 fixes a crash in the premium voice and makes first-run polish land the way it should.

## Fixed

- **A premium-voice error no longer crashes the app.** A transient inference error inside the on-device voice engine could abort the whole process mid-narration. The engine now reports the failure, Attaché retries with a fresh engine, and if the voice still cannot speak it falls back to an on-device system voice with a visible note. Never silence, never a crash.
- **Long technical strings are never read aloud.** A raw URL or checksum in an agent update used to be spelled out character by character in some paths. Spoken updates now say "a link", "a checksum", or "an ID" while the full detail stays on the card.

## Changed

- **Previews are always in character.** "Preview personality" speaks a short self-introduction written by the personality itself, naming itself in its own style, capped near eight seconds. When no model is connected the preview says so plainly instead of playing a stand-in line, and a previously generated take still replays.
- **You name every Attaché you make.** Creating or customizing a personality starts with an empty name field, so nothing is ever auto-named for you.
- Onboarding no longer offers personality previews; they live in the personality editor.

## Install

Download `Attache.dmg` below and drag Attaché to Applications, or:

```
brew install --cask danbryan/tap/attache
```

Requires macOS 13 (Ventura) or later on Apple Silicon. Signed with a Developer ID, notarized, and stapled.

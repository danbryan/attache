What to check in this build

This build integrates 19 features and fixes from the INF-347 wave. Here's what changed and exactly how to exercise each one, grouped by where it lives in the app. Start with the flagship items; the rest are quick confirmations.

Every item below passed unit tests and integrated ui-smoke. This is your human confirmation pass on real hardware, the thing automation can't do.

Performance (you should feel these immediately)

  1. Launch speed. Cold-launch the app. It should reach the dock noticeably faster than before (voice-catalog enumeration moved off the launch path; init dropped from ~2.1s to ~0.5s in testing).
  2. Settings → Context → Advanced. Open Settings, go to Context, expand Advanced while a voicemail is playing. It should feel instant now, not sluggish. This was the specific lag you reported.
  3. Single-click character switch. In Settings → Personalities, single-click a character card. It switches immediately, no double-click delay. Double-click still opens the editor... actually now Edit is via the card's Edit control and right-click menu.
  4. New Diagnostics. Settings → About has a "Responsiveness" section showing any main-thread stalls by duration bucket. Should be empty or near-empty in normal use.

Voice picker rebuilt

  1. Open a personality in the editor (Settings → Personalities → Edit / Create). The voice picker is now a searchable, filtered list modeled on the macOS voice picker, not one long flat list.
  2. Try the search field (type a name or language), the engine chips (System / Grok / OpenAI), the quality chips (Premium / Enhanced / Compact), and the language filter. Voices group by language with your locale first.
  3. Each row has a play-sample button. Confirm it previews on click and nothing auto-plays when the picker opens.

Right-click menus, everywhere new

Secondary-click each bottom-bar control. Hold Option to reveal the power/destructive variants.

  1. Settings icon: jumps straight to Appearance / Voice / Personalities / Context / Integrations / Memory. Option adds Diagnostics + Open Support Folder.
  2. Voicemail: Play Recap, Play Latest, Mark All Read, Open Inbox. Option turns Mark All Read into Archive All, and adds Forget Session.
  3. Call: Start Call, Start Private Call, Call as… (personality submenu). Option variants start private.
  4. Personality: Switch Character, Edit, Previous/Next. Option adds Export.

Known limitation: the "Diagnostics" item under Option is disabled (no Diagnostics pane exists yet, it's a placeholder). And on this macOS build, driving these menus via accessibility automation doesn't work, so they're human-tested only, click them yourself.

Privacy incognito

  1. Start Private Call (right-click Call). The character should wear a dark crown band, a PRIVATE chip appears in the call HUD, and the window edge tints. Confirm no history card is written after you hang up.
  2. Make This Call Private mid-call. Start a normal call, then use the HUD action to convert it. Everything already recorded for that conversation is scrubbed; if the scrub fails it says "Still recorded" rather than lying. There is no way back to saved (one-way door, by design).
  3. Private Session (right-click a session row in ⌘K or the inbox): "Don't Record This Session" (shows an incognito glyph) and "Forget This Session…" (states the real card/index counts before deleting).
  4. Echo + Reduce Motion. Switch to the Echo personality, turn on System Settings → Accessibility → Reduce Motion. Echo's voice bars should calm down (this was a real bug the wave fixed).

Captions (the checksum problem)

  1. Play a card whose text contains a long checksum, a URL, or a long identifier. The oversized token should wrap or scale instead of overflowing the caption box, and a long token now highlights progressively across its fragments instead of freezing as one block.
  2. Hover a caption word: an underline plus a m:ss timestamp tooltip appears; click still jumps to that word (unchanged).
  3. Other languages: a personality set to a Korean or Spanish voice should caption that language while the app chrome stays in your UI language.

Keyboard niceties

  1. ⇧⌘P palette: press 1–9 (with the search empty) to switch straight to that personality.
  2. ⌘I inbox and ⌘Y history: j/k move the selection like arrow keys (when the search field isn't focused).
  3. Global summon hotkey: off by default. Set one in Settings, then confirm it brings Attaché forward from another app. Clear it to unregister.

New agent sources

  1. Grok Build and opencode sessions now appear in ⌘K search and the session list (enable them in Settings → Integrations). Grok Build produces live narrated cards; opencode currently does search/index only (live narration is a known follow-up).
  2. Local-model badge: your claude-oss (Ollama-backed) Claude Code sessions now show a small "local" chip distinguishing them from cloud sessions.
  3. Session titles: rename a Claude Code session in the CLI mid-session, and Attaché should pick up the new name (this fixes the "local-command-caveat" title bug you hit).
  4. Subagent activity: when a watched session delegates to subagents, you should now get narration/activity during their work instead of silence.

Two things intentionally not finished

  1. Summarize Session appears in session right-click menus (⌘K, inbox, history) but is currently a no-op: the engine is built and safe, but the cost-preview sheet that runs it wasn't built yet. Clicking it does nothing on purpose. This is the top follow-up.
  2. Kokoro voice engine (INF-366) was never built, dropped from this wave. Not in this build.
